# 能力模型（Competency Model）

## 一、模块说明

本模块构建了《数字逻辑与数字电路》课程的能力模型，明确课程在新工科背景下应培养的核心工程能力，并建立知识、能力与实践任务之间的结构化映射关系。

本能力模型服务于教学设计、学习路径规划与学情诊断，是实现“以学生能力发展为中心”的关键基础。

---

## 二、设计理念

传统教材侧重知识体系组织，而本课程在此基础上进一步构建能力导向体系，将“学什么”转化为“会做什么”，推动教学从知识传授向能力培养转变。

通过能力模型的引入，课程实现了：

- 知识结构 → 能力结构的映射  
- 教学内容 → 工程任务的对接  
- 学习过程 → 能力发展的可观测  

---

## 三、能力结构

本课程能力体系分为三个层级：

### 🟢 基础认知能力

- 理解数字逻辑基本原理  
- 掌握组合逻辑与时序逻辑基础  
- 理解电路设计基本方法  

### 🟡 设计实现能力

- 设计组合逻辑电路  
- 设计时序逻辑系统  
- 进行模块化系统设计  

### 🔴 工程实践能力（重点）

- 使用 FPGA 实现数字系统  
- 运用硬件描述语言进行开发  
- 完成系统调试与优化  

---

## 四、知识—能力—任务映射

本课程建立了知识点、能力与实践任务之间的映射关系：

| 知识点 | 对应能力 | 实践任务 |
|--------|----------|----------|
| 组合逻辑 | 逻辑设计能力 | 编码器 / 译码器设计 |
| 时序逻辑 | 时序分析能力 | 计数器设计 |
| 状态机 | 系统设计能力 | 交通灯控制系统 |
| FPGA开发 | 工程实现能力 | 数字系统实现项目 |

该映射关系使教材内容、习题系统与工程实践形成有机统一。

---

## 五、在教学系统中的作用

能力模型在本课程教学系统中发挥以下作用：

- 支撑动态诊断系统进行能力维度分析  
- 支撑路径规划系统进行个性化任务推荐  
- 指导教材内容组织与实践任务设计  
- 实现学生能力发展过程的持续跟踪  

---

## 六、与其他模块的关系

本模块与其他模块形成协同关系：

- 与 `knowledge-graph/`：建立知识与能力的映射关系  
- 与 `exercises/`：支撑习题按能力维度分类  
- 与 `projects/`：对接工程实践能力培养  
- 与 `tools/`：作为智能诊断与路径规划的决策依据  

---

## 七、总结

本能力模型实现了课程由“知识导向”向“能力导向”的转型，是构建教学闭环与实现个性化教学的重要基础。
